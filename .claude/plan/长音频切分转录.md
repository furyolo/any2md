# 长音频切分转录功能实施计划

## 1. 目标与边界

### 目标
为 any2md 的本地音频转录功能（`--audio-backend qwen-local`）增加长音频自动切分能力，解决单次转录时长限制问题。

### 功能边界
1. **切分策略**：
   - 检测音频时长，超过阈值（默认 10 分钟）自动切分
   - 使用 ffmpeg 按时间段切分，避免加载整个音频到内存
   - Phase 1 简单按时间切分（不对齐静音区间）

2. **转录流程**：
   - 顺序转录各切片（避免并发导致资源竞争）
   - 每个切片独立调用 Qwen3-ASR
   - 失败切片支持重试（最多 3 次）

3. **结果合并**：
   - 按切片顺序拼接文本
   - 切片间添加换行符分隔
   - 保留原有后处理逻辑（t2s 等）

4. **进度显示**：
   - CLI 输出：`[1/5] 转录切片 1... 完成 (12.3s)`
   - 显示当前切片序号、总数、耗时
   - 失败时显示重试次数

### 技术约束
- **仅影响 qwen-local 后端**：AUC 后端由服务端处理长音频，无需客户端切分
- **保持现有接口**：AudioConverter 签名不变，切分逻辑封装在内部
- **临时文件管理**：切片文件使用 TemporaryDirectory，自动清理
- **依赖工具**：ffmpeg（已有依赖）

### 配置参数
- `--qwen-chunk-duration`：切片时长（秒），默认 600（10 分钟）
- 环境变量：`ANY2MD_QWEN_CHUNK_DURATION`

### 验收标准
1. 15 分钟音频自动切分为 2 个切片，顺序转录并合并
2. CLI 显示进度：`[1/2] 转录切片 1... 完成 (45.2s)`
3. 单个切片失败时重试，全部失败时报错
4. 短音频（< 阈值）不切分，行为与现有逻辑一致
5. 临时切片文件自动清理

### 非目标（后续 Phase）
- 智能切分点检测（静音区间对齐）
- 并发转录多个切片
- 断点续传

---

## 2. 架构决策

1. 只改 `qwen-local` 且只在 `settings.runtime == "qwen-asr"` 时启用切片逻辑
2. 保持 `AudioConverter` 和 `QwenAsrAudioConverter.__call__(path)` 的外部调用方式不变，把切片、重试、进度都封装在 `QwenAsrAudioConverter` 内部
3. 使用 `ffprobe` 探测时长，使用 `ffmpeg` 按固定时长切片，临时切片放进 `TemporaryDirectory`，顺序处理，处理完自动清理
4. 默认不并发，避免 GPU/CPU/内存争抢
5. 重试粒度是"单切片"，最多 3 次；单片最终失败则整体失败
6. 进度输出通过 CLI 注入一个本地回调，AUC 现有进度回调不受影响

```python
def _probe_audio_duration(path: Path) -> float:
    """使用 ffprobe 探测音频时长（秒）"""
    cmd = [
        "ffprobe",
        "-v", "error",
        "-show_entries", "format=duration",
        "-of", "default=noprint_wrappers=1:nokey=1",
        str(path)
    ]
    result = subprocess.run(cmd, capture_output=True, text=True, check=True)
    return float(result.stdout.strip())
```

### Step 4: 实现音频切分
**文件**：`src/any2md/converters/audio.py`

新增函数：
```python
def _split_audio_chunks(
    source: Path,
    chunk_duration: int,
    total_duration: float,
    output_dir: Path
) -> list[Path]:
    """使用 ffmpeg 按时间切分音频"""
    chunks = []
    num_chunks = int(total_duration / chunk_duration) + 1

    for i in range(num_chunks):
        start = i * chunk_duration
        chunk_path = output_dir / f"chunk_{i:03d}{source.suffix}"

        cmd = [
            "ffmpeg",
            "-i", str(source),
            "-ss", str(start),
            "-t", str(chunk_duration),
            "-c", "copy",
            "-y",
            str(chunk_path)
        ]
        subprocess.run(cmd, check=True, capture_output=True)
        chunks.append(chunk_path)

    return chunks
```

### Step 5: 实现切片转录逻辑
**文件**：`src/any2md/converters/audio.py`

修改 `QwenAsrAudioConverter.__call__()` 方法：
```python
def __call__(self, path: Path) -> str:
    # 1. 探测时长
    duration = _probe_audio_duration(path)

    # 2. 判断是否需要切分
    if duration <= self.settings.chunk_duration_seconds:
        # 短音频，直接转录
        return self._transcribe_single(path)

    # 3. 长音频，切分后转录
    with tempfile.TemporaryDirectory() as tmpdir:
        chunks = _split_audio_chunks(
            path,
            self.settings.chunk_duration_seconds,
            duration,
            Path(tmpdir)
        )

        results = []
        for idx, chunk in enumerate(chunks, 1):
            text = self._transcribe_chunk_with_retry(
                chunk, idx, len(chunks)
            )
            results.append(text)

        return "\n".join(results)

def _transcribe_chunk_with_retry(
    self, chunk: Path, idx: int, total: int
) -> str:
    """转录单个切片，支持重试"""
    max_retries = 3

    for attempt in range(1, max_retries + 1):
        try:
            start_time = time.time()

            if self.progress_callback:
                self.progress_callback({
                    "kind": "chunk_start",
                    "index": idx,
                    "total": total,
                    "attempt": attempt
                })

            text = self._transcribe_single(chunk)

            elapsed = time.time() - start_time
            if self.progress_callback:
                self.progress_callback({
                    "kind": "chunk_completed",
                    "index": idx,
                    "total": total,
                    "elapsed": elapsed
                })

            return text

        except Exception as e:
            if attempt == max_retries:
                raise MediaProcessingError(
                    f"切片 {idx}/{total} 转录失败（已重试 {max_retries} 次）: {e}"
                )

            if self.progress_callback:
                self.progress_callback({
                    "kind": "chunk_retry",
                    "index": idx,
                    "total": total,
                    "attempt": attempt,
                    "error": str(e)
                })
```

### Step 6: CLI 进度回调
**文件**：`src/any2md/cli.py`

在构建 converter 时注入进度回调：
```python
def _build_qwen_local_converter(args, stderr):
    settings = resolve_local_qwen_audio_settings(...)

    def progress_callback(event):
        kind = event["kind"]
        if kind == "chunk_start":
            idx, total = event["index"], event["total"]
            attempt = event.get("attempt", 1)
            retry_msg = f" (重试 {attempt})" if attempt > 1 else ""
            print(f"[{idx}/{total}] 转录切片 {idx}{retry_msg}...", file=stderr)

        elif kind == "chunk_completed":
            idx, total = event["index"], event["total"]
            elapsed = event["elapsed"]
            print(f"[{idx}/{total}] 转录切片 {idx}... 完成 ({elapsed:.1f}s)", file=stderr)

        elif kind == "chunk_retry":
            idx, total = event["index"], event["total"]
            attempt = event["attempt"]
            error = event["error"]
            print(f"[{idx}/{total}] 切片 {idx} 失败，准备重试 ({attempt}/3): {error}", file=stderr)

    if settings.runtime == "qwen-asr":
        return QwenAsrAudioConverter(
            settings,
            progress_callback=progress_callback
        )
    else:
        return LocalQwenAudioConverter(settings)
```

```python
class QwenAsrAudioConverter:
    def __init__(
        self,
        settings: LocalQwenAudioSettings,
        *,
        model_loader: Callable[[LocalQwenAudioSettings], Any] | None = None,
        duration_probe: Callable[[Path], float] | None = None,
        audio_splitter: Callable[[Path, int, float, Path], list[Path]] | None = None,
        progress_callback: Callable[[dict], None] | None = None,
        clock: Callable[[], float] | None = None,
    ):
        self.settings = settings
        self.model_loader = model_loader or _load_qwen_asr_model
        self.duration_probe = duration_probe or _probe_audio_duration
        self.audio_splitter = audio_splitter or _split_audio_chunks
        self.progress_callback = progress_callback
        self.clock = clock or time.time
        self._model = None
```

```python
def test_chunk_retry_on_failure():
    settings = LocalQwenAudioSettings(
        runtime="qwen-asr",
```

## 4. 验收映射

实现完成后按以下顺序验证：

1. **15 分钟音频切分**：`duration_probe > 600`，应切成 2 片，合并后返回完整文本
2. **CLI 进度显示**：stderr 包含 `[1/2] 转录切片 1... 完成 (xx.xs)`
3. **重试逻辑**：某一片第一次失败、第二次成功时，应出现重试日志且最终整体成功
4. **短音频不切分**：`duration < 600` 时，不调用 `audio_splitter`
5. **临时文件清理**：退出 `TemporaryDirectory` 后，切片目录自动清理

---

## 5. 注意事项

- `ffprobe`/`ffmpeg` 缺失时要报清晰错误，不能静默跳过切片
- `-c copy` 速度快，但对少数容器边界可能不稳定；如果后续发现兼容性问题，再在 Phase 2 增加转码到 `.wav` 的兜底分支
- 不要把切片进度和 AUC 进度回调混在一起，它们是两套语义
- 保持 `t2s` 仍在 CLI 输出前执行，避免在切片层重复处理
